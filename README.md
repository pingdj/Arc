1. 软件简介
   
ARC 批量工具是一款专为 ARC 测试网设计的桌面应用，支持：

批量 Mint Activity：自动为多个钱包执行 mintActivity 操作。

自动兑换循环：对每个钱包执行 USDC → EURC → USDC 的兑换循环，模拟真实交易路径。

软件采用一机一码激活机制，每台电脑需要独立的激活码。请通过作者获取。

2. 系统要求
   
操作系统：Windows 10 / 11（64 位）

网络：需要能够访问 ARC 测试网 RPC（https://rpc.testnet.arc.network）

建议：开启代理（国内用户必须），代理地址如 http://127.0.0.1:7897

3. 安装与运行

下载 ARC_Tool.exe 并放置在一个单独的文件夹中。

首次运行：会弹出激活窗口，显示本机机器码。请将机器码复制并发送给作者，作者会返回激活码。

输入激活码，点击「激活」，激活成功后即可正常使用。

软件会最小化到系统托盘，可通过托盘图标显示窗口或退出。

4. 配置文件说明
   
软件在程序同级目录下会自动生成或需要您手动创建以下配置文件：

文件名	作用	格式
private_keys.json	存储钱包私钥（必填）	{"private_keys": ["0x私钥1", "私钥2"]}
proxy_config.json	代理配置（可选）	{"enabled": true, "url": "http://127.0.0.1:7897"}
config.json	时间间隔配置（可选）	{"mint_interval": 3, "swap_interval": 10}
注意：私钥请勿泄露，软件仅本地使用，不上传任何私钥数据。

5. 使用步骤
   
5.1 配置私钥

在软件同级目录创建 private_keys.json，内容示例：

{
    "private_keys": [
        "0x1234567890abcdef...",
        "0xabcdef1234567890..."
    ]
}

私钥可以带 0x 前缀，也可以不带。

每个私钥对应一个钱包地址，软件会依次处理。

启动软件后，点击「重新加载私钥」按钮，界面会显示已加载的私钥数量。

5.2 设置代理（国内必须）

点击「修改代理」或在配置菜单中打开代理设置。

勾选「启用代理」，填写代理地址（例如 http://127.0.0.1:7897）。

保存后，正在运行的任务会自动重启，代理立即生效。

5.3 调整时间间隔（可选）

Mint 间隔：每个钱包执行 Mint 后的等待秒数（默认 3 秒）。

Swap 间隔：每个钱包完成一次完整兑换循环后的等待秒数（默认 10 秒）。

可在「修改时间间隔」中调整，保存后正在运行的任务会自动重启。

5.4 启动任务

Mint 任务：点击「启动 Mint」，软件会按顺序为所有私钥调用 mintActivity 方法。日志窗口会实时显示提交的交易哈希。

Swap 任务：点击「启动 Swap」，软件会对每个钱包依次执行：

从钱包余额中随机抽取 0.5~2 USDC（原生代币）兑换成 EURC。

等待若干秒（由间隔决定）后，将收到的 EURC 全部兑换回 USDC。

循环结束后等待间隔时间，再处理下一个钱包。

两个任务可以同时运行，互不干扰。

5.5 停止任务

点击「停止 Mint」或「停止 Swap」，软件会在当前钱包操作完成后停止（不会立即中断交易发送，但会停止下一个钱包的执行）。

5.6 查看日志

每个任务有独立的日志窗口，显示执行状态、交易哈希、错误信息等。

可点击「清空日志」清除当前窗口内容。

6. 常见问题
   
Q1：激活码如何获取？

A：首次运行软件会弹窗显示本机机器码。请将机器码复制后发送给作者（邮箱或其它联系方式），作者会返回激活码。输入激活码即可激活。

Q2：激活后，软件会收集我的隐私吗？

A：不会。软件仅在本机读取私钥配置文件，所有交易签名都在本地完成，不上传任何私钥或敏感数据。激活机制仅验证机器码，不收集个人信息。

Q3：Mint 或 Swap 失败怎么办？

A：请检查：

钱包地址是否有足够的原生代币（用于支付 gas 和兑换）。

代理是否已正确配置（国内必须）。

RPC 是否可访问（可尝试更换节点，目前固定为 https://rpc.testnet.arc.network）。

日志中会显示具体错误原因，如 insufficient funds 或 timeout。

Q4：可以自定义 RPC 吗？

A：当前版本不支持，RPC 地址硬编码在程序中。如有需要，可联系作者定制。

Q5：软件支持多开吗？

A：不支持。软件有单实例检测，只能运行一个进程。

Q6：杀毒软件报毒怎么办？

A：由于软件使用 PyInstaller 打包，可能被部分杀毒软件误报。请添加至信任区或暂时关闭杀软。代码完全开源（如果您获取了源码），可自行审查。
